面向对象之原型

原型链

  • 什么是原型链？
    • 原型链是由对象组成的有限对象链，用于实现继承和共享属性。
  • 原型链的作用？
    • 实现数据共存、继承
  • 原型链的本质？
    • 实例对象与原型形成的链，链之间的联系通过proto。
  • 没有办法通过实例修改原型上的内容。
  • 当要找的某个属性找到了，就不会再向上找了，如果没找到就会按着原型链继续往上找直到找到。

继承的思想：别人给了我，我的还是我的

函数

  • 函数扮演的角色：(函数扮演什么角色取决于函数的使用方式)
- 普通函数
- 构造函数
- 函数也是对象（是函数也是对象 jquery里内部体现了这两种角色
 $('div') $是函数    $.ajax(); $是对象）

  • 其中Math不是函数是对象

    • console.log( typeof Math); object
    • console.log(typeof Array); function
    • console.log(Math instanceof object) true Math是Object的实例

  • 函数的种类

    • 系统函数（内置API）
    • 自定义函数（主讲）

  • 定义方式：

    • 函数声明
      function fn(){}
    • 函数表达式
      var fn = function(){}
    • Function (很少有 在框架内部会用)
      var fn = new Function(参数，函数体) 最后一个参数是函数体，前后的参数都是对应的形参

函数声明和函数表达式在预解析时有差别前者调用可以在声明前后者不可以，在功能上无差别

  • 浏览器的解析器在运行JS代码的时候，分为两个阶段：

    • 预解析 函数声明、变量声明、参数声明会提升
    • 执行

  • JS中函数可以先使用后定义吗？答：可以，因为预解析会将函数的声明提前

  • 函数的调用

    • 普通函数的调用
    • 构造函数的调用
    • 对象方法的调用
      • call/apply的调用

  • 函数的调用方式觉得了this的指向（5种）

    1)普通函数里的this
    2)构造函数
    3)原型方法中的this和构造函数中的this指向相同，都是实例对象
    4)对象的方法
    5)定时函数
    如果作为普通函数使用，不推荐在函数内使用this,在严格模式下，普通函数中的this就是window
