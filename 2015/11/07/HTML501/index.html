HTML5初识

认识HTML5

  • HTML5是WHATWG和W3C博弈后的结果，它的牵头人是WHATWG.
  • HTML5简称H5.
  • 狭义HTML5:html5新的语义化标签，表单元素等。
  • 广义HTML5：HTML5+CSS3+JS
  • HTML5不仅仅是html的最新的一个版本，而且它制定了一系列的应用开发标准，是将web应用到应用开发平台的第一个HTML语言。
  • HTML5引进了一些新的元素，例如：语义化标签，智能表单等,方便我们创建富互联网应用制定了一系列的javascriptAPI，例如：地理定位，重力感应等。还可以结合canvas开发网络游戏。

语法规范

  1. 更加宽松
  2. 更加简洁
  3. 在实际开发中，应规范书写，不要太随意。

什么是语义化标签

  • 赋予标签意义，用合适的标签描述web结构。

  • 常用的语义化标签

    • header 头部标签
    • nav 导航标签
    • section 区域标签 （sect + ion 分+不同）
    • article 文章标签
    • aside 侧边栏标签
    • footer 页脚标签

  • section与div的区别

    • section比div更具有语义化

  • 使用新的语义化标签的好处

    • 对搜索引擎（百度，goole）友好。
    • SEO(search engine optimizer)搜索引擎优化
    • 有了良好的结构和语义的网页内容更容易被搜索引擎抓取，且获取到更高的权重。在用户搜索相关内容时，排名会靠前，自然会获得更高的流量。
    • 新的语义化标签从IE9开始兼容。

时间、媒体区域和标记标签

  • time 时间标签
  • <time>2015年11月21日</time>
  • time标签里又提供了一个属性 datetime,这个属性指明了具体的时间
  • figure 媒体区域标签，用来表示媒体（图片，视频，音频）
  • figcaption 媒体区域的标题
  • mark 标记标签 会产生荧光笔的效果

进度条标签

  • progress 进度条标签
    • <progress min="0" value="20" max="100"></progress>

计量标签

  • meter 计量标签（可以认为是progress的升级版，多了两个属性：low high.）
    • 当value<=low 的时候，呈现绿色；
    • 当value>low且value<=high的时候，呈现黄色；
    • 当value>high的时候，呈现红色。
    • <meter min="0" max="100" value="60" low="70" high="80"></meter>

表单(带缓存，如果按F5是无法刷新表单的，按ctrl+F5用来强制刷新表单，或者重新打开页面)

  • 随着互联网应用和移动开发的发展，传统的表单越来越不能满足我们的需要了。HTML5对表单进行了调整
  • 新的表单元素类型
    • 邮箱类型：type=”email”
    • 拾色器： type=”color”
    • 网址： type=”url”
    • 数字： type=”number”（值可以输入数字，可以输入e科学计数法）
    • 范围： type=”range”
    • 搜索框： type=”search”（与text暂时是一样的）
    • 电话号码： type=”tel”
    • 日期： type=”date”(有日期组件)
    • 时间： type=”time”
    • 日期和时间：type=”datetime”(通常浏览器不支持)
    • 周数： type=”week”
    • 月： type=”month”
    • 本地日期和时间：type=”datetime-local”
    • <fieldset></fieldset>表单框
    • <legend></legend>表单标题
  • 新增的表单元素
    • datalist 数据列表
    • datalist是隐藏在表单域中的，不显示的，使用它里面数据的表单元素应该用list属性绑定datalist的id
    • output 输出
    • <output></output> 用于输出计算结果的

  • 新的表单属性

    • 自动获取焦点： autofocus（常用）
    • 必填项： required（常用）
    • 自动完成： autocomplete（常用）可写在form上（使表单域下的所有表单元素都会自动完成）
    • 禁止自动完成： autocomplete=”off”
    • 占位符： placeholder=”请输入用户名”
    • 正则表达式： pattern=”正则表达式” []表示选择的范围 {}表示选择几个
    • 多个邮箱： multiple(允许输入多个邮箱或file)逗号分割
    • 多个文件： multiple
      -表单域外的一个元素：form=”表单域的id”在表单提交的时候将它也一起提交，表示该表单元素属于这个表单域。

  • 新的表单事件

    • oninvalid 表单元素验证不通过时，触发

    • oninput 当用户输入时，触发

    • oninput与onchange的区别

      1. oninput在用户一输入时就触发，onchange是用户输入后，并且该表单元素失去焦点时才触发。
      2. oninput只能作用于input和textarea;onchange可以作用于input，textarea和select等
      3. oninput的兼容性没有onchange好。

属性

  • autoplay autoplay 如果出现该属性，则视频在就绪后马上播放。
  • controls **controls 如果出现该属性，则向用户显示控件，比如播放按钮。
  • height **pixels 设置视频播放器的高度。
  • loop loop 如果出现该属性，则当媒介文件完成播放后再次开始播放。
  • muted muted 规定视频的音频输出应该被静音。
  • poster URL 规定视频下载时显示的图像，或者在用户点击播放按钮前显示的图像。
  • preload preload 如果出现该属性，则视频在页面加载时进行加载，并预备播放。如果使用 “autoplay”，则忽略该属性。
  • src url 要播放的视频的 URL。
  • width pixels 设置视频播放器的宽度。
